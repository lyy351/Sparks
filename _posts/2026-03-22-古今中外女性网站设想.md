## 各文件/文件夹详细说明

### 1. 根目录核心文件
- **`config.toml`**：Hugo站点配置，包括网站标题、语言、主题、菜单、自定义参数等。你需要在这里定义主菜单（现实人物、神话传说、专题），并启用自定义模板路径。

### 2. `content/` – 所有人物文章
- 按“现实 vs 神话”分成两大目录，再按地域/文化进一步细分。  
- 每个人物是一个 `.md` 文件，在文件头部用 Front Matter 填写标签、分类、图片等。  
- 脚注使用标准 `[^1]` 格式，Hugo会自动生成脚注列表，再由模板提取到右侧栏。

### 3. `layouts/` – 控制页面显示
- **`_default/baseof.html`**：基础骨架，包含 `<html>`、`<head>`、`<body>` 以及 `{{ block }}` 占位符。
- **`_default/single.html`**：人物详情页模板。  
  - 这里实现 **左文右证** 布局：左侧 `.main-content` 渲染 `{{ .Content }}`，右侧 `.references` 用来显示参考文献。  
  - 需要编写脚本来提取脚注并动态高亮（可直接在模板内嵌JS或引用外部 `static/js/references.js`）。
- **`_default/list.html`**：列表页（如 `/reality/china/modern/`）模板。展示人物卡片网格，支持分页和排序。
- **`partials/`**：可复用组件，例如：
  - `header.html`：包含网站logo、主菜单、搜索框（可用Pagefind）。
  - `footer.html`：版权、来源声明等。
  - `sidebar.html`：右侧边栏（标签云、随机推荐）。注意人物详情页右侧已被参考文献占，侧边栏可放在左侧下方或单独处理。

### 4. `static/` – 静态资源
- **CSS**：控制两栏布局、响应式、脚注样式等。  
- **JS**：实现右侧参考文献动态高亮、移动端折叠面板等。  
- **images**：按人物分类存放，使用相对路径引用，如 `/images/china/ancient/wangzhenyi/portrait.jpg`。

### 5. 主题与自动部署
- 如果你不想从零写模板，可以选用一个现成主题（如 `hugo-theme-stack`），然后在其基础上修改 `single.html` 实现两栏布局。  
- **`.github/workflows/hugo.yml`**：可选的自动化部署配置，让你每次 `git push` 后，GitHub自动构建并发布到 GitHub Pages。Hugo官方有现成的工作流模板。

1. **先搭骨架**  
   在你的电脑上创建上述文件夹结构（可暂时忽略 `themes/` 和 `.github/`），并创建 `config.toml` 和一个测试人物（如 `content/reality/china/ancient/wangzhenyi.md`）。  
2. **写一个简单模板**  
   在 `layouts/_default/single.html` 中先用最简单的单栏布局输出 `{{ .Content }}`，确保站点能运行。

5. **部署到 GitHub**  
   将仓库推送到GitHub，开启Pages（如果使用自动构建，则直接推送即可）。

# 网站构想存档：《女》

## 一、核心理念
让那些被历史淹没的女性故事重新浮出地表。  


## 二、内容范围
- **现实人物**：古今中外女性，按地域（中国、亚洲其他、欧洲、美洲、非洲/大洋洲）和时代（古代、近现代、当代）分类。
- **神话传说**：中国神话、希腊神话、北欧神话、其他神话/宗教中的女性神祇与传说人物。
- **专题策展**：如《山海经》女神、诺贝尔奖女性、民国女报人、各国“第一位”女性等。

## 三、页面功能与布局

### 1. 人物详情页
- **左栏**：生平小传（800–1200字），用脚注标注引用来源。
- **右栏**：同步显示参考文献原文（引文、出处、页码），滚动时高亮当前脚注对应的原文。
- **基本信息**：头像、生卒年、朝代/时代、领域、身份标签。
- **关联人物**：亲属、师友、同时代相关者。
- **名言/作品**：精选。
- **延伸阅读**：推荐书籍、纪录片、数据库链接。
- **资料来源**：注明引用来源。

### 2. 首页
- 英雄区（slogan：“穿越古今神话，看见她们的故事”）
- 数据统计（已收录人物总数、覆盖地域/神话体系数量）
- 最新添加（按时间倒序）
- 随机推荐（6–9位，避开最新）
- 热门专题入口（图标+文字）

### 3. 列表页
- 按分类展示人物卡片（头像、姓名、生卒、一句话简介、标签）
- 支持分页和简单排序

### 4. 专题页
- 专题引言 + 特色内容（如时间轴、数据图表、关系网络图）
- 该专题下的人物列表
- 延伸资料链接

### 5. 技术呈现
- 左文右证布局（HTML/CSS/JS 实现脚注与参考文献的联动高亮）
- 标签云、随机推荐、搜索（前端实现）
- 关系网络可视化（可选，用 d3.js）

## 四、技术架构细节（基于 Hugo）

### 1. 人物专属编码（ID）方案
- 格式：`<拼音标识>-<生卒年或时代>`  
  示例：`wangzhenyi-1768-1797`、`xiwangmu-myth-china`
- 编码作为文件名，确保唯一性，同时 URL 中可直接体现人物信息。

##五、技术实现历程与放弃原因

### 1. 本地构建成功，但产物无效
在本地运行 `hugo` 命令，输出显示 `Total in 92 ms` 且统计出 38 个页面，但生成的 `public/index.html` 实际内容为 **0 字节**。尝试用 `hugo server` 预览同样空白。  
可能原因：模板中某些 partial 缺失、变量未定义、或配置与 Hugo 版本不兼容。由于排查成本过高，未能修复。

### 2. 部署尝试全部失败
- **GitHub Actions 自动构建**：国内网络下频繁失败，或虽显示绿勾但构建产物为空。Actions 日志中无有效错误提示。
- **手动上传 `public` 内容**：将 `public` 文件夹内所有文件上传到仓库根目录，设置 Pages 源为根目录后，访问网站显示 RSS XML（因 `index.html` 为空，Pages 自动回退到 `index.xml`）。多次尝试删除重建均无效。

### 3. 转向 Jekyll 未果
因 GitHub Pages 原生支持 Jekyll，尝试寻找功能接近的 Jekyll 主题。但搜索五六个网站，要么国内访问不了，要么功能过于简陋（无法实现左文右证、复杂标签等）。即使提出可手动创建极简 Jekyll 网站，你也因对“从0到1”模式失去信任而放弃。

### 4. 更换 Hugo 主题受挫
选定 `cncf/dot-org-hugo-theme` 主题，但其安装需要 `git clone`、`npm install`、本地构建，且国内访问 GitHub 极不稳定，无法完成。

### 放弃原因总结
- **工具与需求错配**：设计需要 Hugo 的强大模板能力，但希望完全在 GitHub 网页上完成（无需本地操作）。Hugo 无法满足后者，Jekyll 无法满足前者。
- **网络限制**：GitHub Actions 在国内极不稳定，手动上传又因本地生成产物无效而失败。
- **调试成本过高**：本地构建“成功”但输出为空，问题根源难以定位，耗费大量时间后未能解决。
- **心力耗尽**：在反复尝试不同方案均告失败后，决定暂停项目，将现有构想与内容存档。
